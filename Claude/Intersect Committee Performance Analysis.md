

Critical, evidence-based performance reports on Intersect MBO committees, generated from meeting transcripts and summaries using the Anthropic API.

## Quick Start

```bash
# 1. Install dependency
pip install anthropic

# 2. Set API key
export ANTHROPIC_API_KEY="sk-ant-..."

# 3. Copy these files into your vault
cp committee_analysis.py config.py prompts.py \
   ~/Desktop/Intersect\ Research/scripts/

# 4. cd into scripts directory
cd ~/Desktop/Intersect\ Research/scripts/

# 5. Dry run to verify file detection
python committee_analysis.py --committee MCC --prompt all --dry-run

# 6. Run MCC first (you have transcripts → highest quality)
python committee_analysis.py --committee MCC --prompt 1
python committee_analysis.py --committee MCC --prompt 2
python committee_analysis.py --committee MCC --prompt 3
python committee_analysis.py --committee MCC --prompt 4
python committee_analysis.py --committee MCC --prompt 5

# 7. Or run all prompts at once
python committee_analysis.py --committee MCC --prompt all
```

## The 5 Prompts

|#|Name|What it does|Needs transcripts?|
|---|---|---|---|
|1|Summaries vs Transcripts|Forensic truth audit: compares each summary to its transcript, finds fabrications, omissions, sanitised conflict|**YES** — skips if none|
|2|Member Participation|Attendance grid + deep per-member analysis: speaking, contributions, between-meeting work, accountability, ranking|Best with transcripts, works with summaries|
|3|Leadership Assessment|Chair / Vice Chair / Secretary evaluation: facilitation, decision quality, boundary respect, strategic direction|Best with transcripts, works with summaries|
|4|Promises vs Reality|Tracks every commitment against delivery: delivery rate, biggest failures, most/least reliable members|Best with transcripts, works with summaries|
|5|Critical Observations|Free-form governance report: red flags, cultural dynamics, power plays, community engagement, recurring failures|Best with transcripts, works with summaries|

## How It Works

### Prompt 1 (Truth Audit)

Runs in two phases:

1. **Per-meeting**: One API call per Summary+Transcript pair. The model gets full attention on each meeting.
2. **Synthesis**: All per-meeting findings are combined and a final API call identifies cross-meeting patterns.

### Prompts 2-5 (Bulk Analysis)

1. Tries to fit ALL files in a single API call.
2. If content exceeds the context window, auto-chunks by meeting date.
3. Each chunk is analysed separately, then a synthesis pass merges the results.

### Audit Trail

Every API call creates three files:

- `*_meta_in.json` — timestamp, model, token estimates
- `*_full_input.txt` — exact system + user prompt sent to the API
- `*_meta_out.json` — actual token counts, elapsed time, stop reason

This means any finding can be verified by checking what the model actually saw.

## Configuration

Edit `config.py` to change:

- `VAULT_ROOT` — path to your Intersect vault
- `MODEL` — default model (override with `--model`)
- `MAX_CONTEXT_TOKENS` — context window budget
- `MAX_OUTPUT_TOKENS` — increase if outputs are truncated
- `RATE_LIMIT_DELAY` — seconds between API calls

### Opus 4 ($15/$75 per M input/output tokens)

Multiply Sonnet costs by ~5x. Use selectively on the most important committees.

## Recommended Workflow

1. **Validate on MCC first** — you have manual reports to compare against
2. **Review Prompt 1 output** — verify the truth audit catches what you caught manually
3. **Run remaining prompts** on MCC, compare quality
4. **If satisfied**, run all prompts on committees with transcripts
5. **Then run** on summaries-only committees (with the caveat noted in output headers)
6. **For publication**, re-run key committees with Opus

## Troubleshooting

**"Output TRUNCATED"** → Increase `MAX_OUTPUT_TOKENS` in `config.py`

**"Input exceeds limit"** → The script will auto-chunk, but if a single meeting's transcript is >180K tokens, you may need to split it manually

**API rate limits** → Increase `RATE_LIMIT_DELAY` in `config.py`

**Wrong vault path** → Edit `VAULT_ROOT` in `config.py`

**Empty files** → The script skips empty files (e.g. placeholder transcript files with 0 bytes)
