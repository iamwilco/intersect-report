# Standard Operating Procedures

Documented solutions to resolved issues. Check here before debugging — someone may have already solved it.

## How to add an SOP

When you resolve a tricky issue, create a file:
```
SOPs/
└── category/
    └── issue-name.md
```

Each SOP should include:
1. **Problem** — What went wrong
2. **Root cause** — Why it happened
3. **Solution** — Step-by-step fix
4. **Prevention** — How to avoid it next time
